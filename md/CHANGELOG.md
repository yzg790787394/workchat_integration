# 更新说明：企微通 (WorkChat) 2026 异步重构版

## 本项目迎来了一次“脱胎换骨”的底层重构。我们彻底重写了核心通信引擎，并在用户体验和诊断工具上引入了多项创新功能。

### 🌟 核心改进：全异步架构 (Full Async Engine)
- 性能飞跃：彻底移除同步阻塞库 requests，全面采用 aiohttp 异步框架。在高频推送或弱网环境下，不再触发 Home Assistant 的系统卡顿警告。
- 并发保护：引入 asyncio.Lock 令牌锁，在高并发场景下有效防止重复请求 Access Token 导致的接口异常。
- 符合 2026 规范：代码全面适配 HA 最新 runtime_data 存储模式，大幅提升了系统的稳定性与内存管理效率。

### 🛠️ 新增功能：Web 状态诊断页 (Diagnostic Web Page)
- 可视化 Webhook 验证：现在直接在浏览器访问你的 Webhook 回调地址，即可进入实时诊断页面。
- 即时诊断：支持实时检测 Access Token 有效性、显示 Agent ID 及其外部访问 URL。
- 通信追踪：新增“最后收信时间”记录，帮助你精准判断内网穿透或企微后台配置是否生效。

### 🖼️ 传感器增强：视觉化与 UI 优化
- 图片直显：图片消息传感器现在支持 entity_picture。当企微收到照片时，HA 仪表盘将直接展示缩略图，点击即可查看大图。
- UI 智能截断：针对超长的 Media ID 字符串，UI 界面现在会进行缩略显示（如 3VHr8...bl6I5g），彻底解决 UI 布局被撑破的问题，同时在属性中完整保留原始 ID 供自动化调用。
- 诊断分类：将回调状态、上传 ID 等辅助实体划归为 DIAGNOSTIC 类别，保持主界面的清爽。

### 📢 通知进化：Notify 实体化
- 全新通知平台：适配最新的 Notify Entity 规范。企微通知现在作为一个独立的实体存在，支持在 HA UI 中进行管理和翻译。
- 双重动作支持：同时支持标准 notify.xxx 实体动作与集成的专属动作 workchat_integration.notify，兼顾标准性与灵活性。

### 🔒 安全与健壮性
- 代理 URL 校验：配置流中新增 Proxy URL 格式自动检测，防止因格式错误导致的集成崩溃。
- Token 容错：增加了对 40014 (Token 过期) 错误码的自动捕获与瞬间重试机制。
- 命名规范化：修复了通知实体在翻译失效时的命名退化问题，确保所有实体遵循“设备名 + 功能名”的统一格式。

### ⚠️ 破坏性变更 (Breaking Changes)
- 最低版本要求：本版本要求 Home Assistant 核心版本 2024.4.0 或更高。
- 配置重新加载：由于存储结构从 hass.data 迁移至 runtime_data，升级后建议删除集成并重新添加，以确保所有新特性生效。
